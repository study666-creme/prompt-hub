# 浏览器插件（规划）

> 状态：**未开发** · 本文供产品评估与后续实现参考。

## 目标

在任意网页上悬浮一个小面板：

| 区域 | 行为 |
|------|------|
| 上方 | 拖入/粘贴图片 → 作为卡片配图 |
| 中间 | 提示词输入框 |
| 下方 | 长条「保存」+ 小号「清空」（防误触） |
| 划选网页文字 | 出现「复制到提示词」按钮 → 填入输入框 |

保存后：卡片写入 Prompt Hub 仓库（需登录），面板清空内容但**保持打开**。

## 技术可行性

**可以实现。** 推荐架构：

1. **Chrome/Edge 扩展**（Manifest V3）：`content script` 注入悬浮 UI + 划词按钮；`service worker` 处理 API。
2. **与主站通信**：扩展打开 `https://prompt-hub.cn` 的登录页或 OAuth，拿到 Supabase session / API token 后调用现有 `POST` 云同步接口（与网页版同一套 API）。
3. **图片**：拖入文件 → `FileReader` / 上传至现有 storage 路径，再创建卡片 JSON。

注意：需新增少量 **扩展专用 API** 或复用 `user_data` 增量写入，避免与网页版全量 merge 冲突。

## 法律与合规（非律师意见，供参考）

| 风险点 | 说明 |
|--------|------|
| **复制网页文字** | 用户手动复制公开可见内容用于个人提示词，一般风险较低；**勿**批量抓取、绕过付费墙或违反网站 ToS。 |
| **图片** | 用户应拥有或有权使用所保存图片；扩展不应鼓励下载受版权保护的他人作品。 |
| **隐私** | 扩展应明确隐私政策：哪些数据上传、仅存用户账号；遵守 GDPR/个保法（若面向 EU/中国用户）。 |
| **商店审核** | 上架 Chrome Web Store 需说明权限用途（`activeTab`、host 权限尽量最小化）。 |
| **目标网站 ToS** | 部分站点禁止自动化提取内容；插件应标注「请遵守所浏览网站的使用条款」。 |

**结论**：作为**个人效率工具**、用户主动操作、数据进自家 Prompt Hub，通常可行；需写好用户协议与权限说明，避免自动爬取与侵权用途。
