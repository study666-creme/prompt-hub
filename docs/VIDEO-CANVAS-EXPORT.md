# 视频画布导出规范（LibTV / TapNow / UpDream）

> 更新：**2026-06-05**  
> 资产创作（`asset-studio.html`）定位：**分镜/视频脚本 + 关联参考图绑定**——前期制作，不替代画布。

---

## 1. 产品定位

| 环节 | 平台 | 做什么 |
|------|------|--------|
| **前期制作** | **Prompt Hub** | 设定库、分镜脚本、提示词与参考图绑定；减轻「对表、找词、找图」 |
| **正式制作** | **画布**（LibTV、TapNow、UpDream 等） | 上传素材、手动 `@` 绑定、生成分镜视频 |

Prompt Hub **不替代**画布；导出包是把前期成果**整齐地带进**画布，而不是跳过画布内的 `@` 步骤。

---

## 2. 命名规范（必须）

导出时，**所有文件名与文档名使用用户在 Prompt Hub 里起的原标题**，便于脚本与图片一一对应：

| 来源 | 用作 |
|------|------|
| 文档标题 | zip 根目录名、`script.md` 内一级标题、可选 `refs.json` 的 `docTitle` |
| 卡片标题 | `images/` 下图片文件名（见下方规则） |
| **禁止** | UUID、`IMG_001` 等无意义代号作为**对外**文件名 |

**图片文件名规则**

- 基础名 = 卡片标题（去非法字符 `\ / : * ? " < > |`，空格可保留或换成 `-`）
- 同标题多张图：追加 `-2`、`-3`…
- 扩展名保留原图格式（`.jpg` / `.png` / `.webp`）

**示例**

- 文档标题：`第三章-雨夜追逐`
- 卡片标题：`主角-林晓`、`参考图-霓虹巷`
- 导出图片：`images/主角-林晓.jpg`、`images/参考图-霓虹巷.png`

脚本正文里引用图片时，使用与 `images/` **相同的显示名**（见 `refs.json` 的 `displayName`），方便用户在画布里 `@` 时辨认。

## 3. 导出 zip 结构

```
{文档标题}.zip
├── script.md          # 分镜/视频脚本文本（含镜头、对白、@占位说明）
├── refs.json          # 图词绑定元数据（机器可读）
├── images/            # 关联参考图，文件名 = 卡片标题
│   ├── 主角-林晓.jpg
│   └── 参考图-霓虹巷.png
└── 导入说明.txt       # 面向小白的逐步操作（可复制进产品 UI）
```

### `script.md`

- Markdown 格式，标题为文档原标题
- 每个镜头/段落可注明需 `@` 的角色或参考图（用 `displayName`，与 `images/` 一致）
- 示例片段：

```markdown
# 第三章-雨夜追逐

## 镜头 1
画面：霓虹巷远景，雨丝。
角色：主角@主角-林晓
参考：场景@参考图-霓虹巷
```

### `refs.json`（示例形状）

```json
{
  "docTitle": "第三章-雨夜追逐",
  "exportedAt": "2026-06-05T12:00:00+08:00",
  "refs": [
    {
      "displayName": "主角-林晓",
      "cardTitle": "主角-林晓",
      "file": "images/主角-林晓.jpg",
      "role": "character"
    },
    {
      "displayName": "参考图-霓虹巷",
      "cardTitle": "参考图-霓虹巷",
      "file": "images/参考图-霓虹巷.png",
      "role": "scene"
    }
  ]
}
```

### `导入说明.txt`（给用户）

简短中文版，与下文「LibTV 手动 @ 流程」一致，可内嵌导出包或帮助页。

## 4. LibTV 手动 @ 流程（不可省略）

**事实**：LibTV、TapNow、UpDream 等画布工具**目前无法在导入 zip 后自动完成 `@` 绑定**。用户必须在画布编辑器里**亲手**完成以下步骤——Prompt Hub 文档与产品文案**不得**暗示可跳过。

### 第 1 步：解压导出包

将 `{文档标题}.zip` 解压到本地文件夹。

### 第 2 步：上传图片到画布

在 LibTV（或同类工具）项目中，把 `images/` 里的文件**全部上传**（可拖拽）。上传后列表里应能看到与 Prompt Hub **相同的文件名**（如 `主角-林晓.jpg`）。

### 第 3 步：打开脚本并对照 `script.md`

在画布中新建或打开分镜/脚本区域，对照 `script.md` 粘贴或分段输入镜头内容。

### 第 4 步：在画布内手动 `@` 每张图

在需要绑定参考图的位置：

1. 输入角色或标签 + `@`，例如：`主角@图片一` 或 `主角@主角-林晓`（以画布实际提示为准）
2. 在弹出的选择器里**点选**第 2 步已上传的那张图
3. 对每个 `refs.json` / `script.md` 里列出的参考图**重复**此操作

**没有捷径**：即使文件名与脚本一致，画布也不会自动关联；**手动 `@` 是必经步骤**。

### 第 5 步：按画布流程继续生视频

绑定完成后，按 LibTV / TapNow / UpDream 各自流程生成预览或成片。

## 5. 路线图

### 短期（当前策略）

- 实现 **画布友好导出包**（本节 zip 结构 + 命名规范）
- 目标平台：**LibTV、TapNow、UpDream**（格式通用，靠可辨认文件名降低 `@` 时找图成本）
- 资产创作 UI：**导出画布包** 为主操作

### 长期（流量与合作）

- 与画布平台洽谈 **一键导入** API / 官方导入通道
- 可能支持：zip 直传、预解析 `refs.json`、半自动 `@`（取决于对方开放程度）
- 在此之前，导出包为**唯一可靠**交付方式
