# UI 与交互规范

## 方向

Prompt Hub 是高频创作工具，不是营销落地页。整体风格应高级、克制、安静，参考 Apple 系统工具的层级、留白和反馈，但不复制品牌资产或用大面积亮蓝色冒充“科技感”。

## 视觉原则

- 主色以黑、白和中性灰建立层级；强调色只表达状态或关键动作。
- 同一页面最多一个高权重主操作，危险操作使用明确警示色。
- 卡片圆角不超过现有设计系统基线；避免卡片嵌卡片和满屏悬浮面板。
- 字号按信息层级固定，不随 viewport 宽度缩放。
- 图标优先使用现有图标库；熟悉操作使用图标按钮和 tooltip。
- 网格背景属于工作区材质，不应降低正文对比度。

## 组件规则

| 需求 | 控件 |
|---|---|
| 模式切换 | segmented control / tabs |
| 二元设置 | switch / checkbox |
| 数值 | slider、stepper 或 number input |
| 颜色 | swatch |
| 工具动作 | 图标按钮 + tooltip |
| 明确命令 | 图标+文字按钮 |
| 多选项 | menu/select |

按钮尺寸、工具栏高度、网格列宽和图片 aspect ratio 要稳定，hover/loading 不得改变布局尺寸。

## 动效

动效用于确认操作和解释状态，不用于持续吸引注意。

- hover/press：100-160ms，轻微颜色、阴影或 scale，按下 scale 不低于 0.98。
- 面板/菜单：180-240ms，opacity + 4-8px 位移，使用 ease-out。
- 新分页卡片：180-260ms，轻微 opacity/translate 缓出，可小范围 stagger。
- 成功反馈：一次性完成，不循环。
- 图片加载：占位到真实图平滑 reveal，不闪白、不改变卡片高度。
- `prefers-reduced-motion: reduce` 下关闭位移、scale 和循环动画。

Masonry/Flex Feed 的卡片入场动画不能写入会影响布局测量的长期 transform，也不能在每次图片 load 后重新播放整墙动画。

## 交互状态

每个异步操作至少覆盖 idle、loading、success、error 和 retry/disabled。加载文案不应挤动按钮；失败要说明下一步，不能只显示“加载失败”。删除、批量下架和数据恢复需要二次确认与影响范围。

## 响应式验收

至少检查 390x844、768x1024、1440x900。确认：

1. 无工具栏挤压、文字截断和不可点击区域。
2. 手机只有 `.app-main` 纵向滚动。
3. 侧栏打开/关闭不引发整页跳动。
4. 键盘焦点清晰，颜色对比满足基本可读性。
5. 动效结束后 DOM 数、卡片位置和滚动锚点稳定。

涉及 UI 的提交应附桌面/手机截图或浏览器验证结果，并说明是否检查 reduced motion。
